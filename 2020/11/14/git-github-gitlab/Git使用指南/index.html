Git使用指南

一、为什么要使用Git版本管理工具

  • 完整的记录项目代码变化的过程

    • 人、时间、地点、物、事

  • 备份每一个变化过程的代码版本

    • 查询历史记录
    • 复原变更
    • 比较差异

  • 多人协同开发

    • 分支合并
    • 克隆提交

  • 需要掌握的知识点：

    • 环境配置
    • 版本追踪
    • GitHub详解
    • 复原变更
    • 分支合并
    • 多人协同开发
    • 服务器自动部署
    • Gitlab详解

二、Git基本使用

I. 环境配置

  • Mac/Linux/Windows 版本下载
  • 测试命令：git –version 查看git版本
  • 常用的bash命令：ls / pwd / clear / touch / mkdir / rm / cd

II. 配置用户名和邮箱[当前项目的追踪]

  • 命令
    • git config –global user.name “processingnewage” [配置全局用户名]
    • git config –global user.email “18955619698@163.com“ [配置全局用户邮箱]
    • git config –list [查看配置]
    • git config user.name [查看用户名]
    • git help [查看帮助]

III 管理git项目

1. 初始化repo和追踪文件、查看状态
  • 命令
    • git init [在项目文件夹中创建.git文件，对项目初始化，实现git的版本管控]
    • git init [name] [在指定的文件夹中生成.git文件，若不存在则创建新的空文件夹]
    • git add [filename] [将文件从workspace添加到cached，追踪文件的变化]
    • git add . / -A [一次性添加所有文件]
    • git status [查看追踪和提交的状态]
2. 放弃追踪
  • 命令
    • git rm –cached [filename] [取消文件的追踪，将cached中的文件回退到workspace中]
3. 提交
  • 命令
    • git commit [对所有被追踪的文件从cached区中添加到本地repo中]
    • git commit -m “描述” [添加描述]
    • git commit -am “描述” [已经add过的文件修改后使用该命令可以不用重新add，直接commit即可]
4. log追踪
  • 命令
    • git log
    • git log -p -2 [比较最近2次提交的差别]
    • git log –author=”processingnewage” [查看作者]
    • git log –oneline [一行显示]
    • git log –graph [线图显示]
    • git log –pretty=oneline
    • git log –pretty=format
5. 追踪文件修改前后的区别
  • 命令
    • git diff [在add之后无法比较]
    • git diff –staged [可以比较在add后的文件，但在commit后的文件始终无法比较]
6. 文件删除、重命名、移动
  • 命令
    • git rm [filename]
    • git mv [oldname] [newname]
    • git mv [filename] demo/demo.txt(folder) [rename and move]
7. 文件忽略(在项目中不是所有的文件都需要commit，因此需要忽略文件)
  • 命令
    • .gitignore [当前目录的根文件夹下]
    • /node_modules [忽略node_modules文件夹下的所有文件]
    • *./log [忽略以.log结尾的所有文件]
    • *.zip [忽略以.zip结尾的所有文件]
    • git rm -r –cached ./[filename] [已追踪的状态无法忽略，需要回退到未追踪的状态][-r 一次性踢出多个文件]
  • tips: 空的文件夹不会被追踪，空文件会被追踪
    .gitignore
/node_modules
*.txt
*.zip
jq.js
8. 项目演练
  • 主要流程
    • git init
    • git add .
    • git commit -m “描述”
  • others
    • git status
    • .gitignore
9. 一键还原
  • 命令
    • git chechout – [filename] [恢复到上一次的状态][未追踪的文件撤销更改]
10. 撤销追踪与文件还原(workspace的文件修改后添加到cached区后，无法直接使用checkout恢复)
  • 命令
    • git reset HEAD [filename] [撤销当前文件的追踪]
    • git chekout – [filename]
11. 版本回退(回到旧版本的同时删除新的版本)
  • 命令
    • git reset –hard HEAD^ [回退到上个版本]
    • git reset –hard HEAD^^ [回退到上上个版本]
    • git reset –hard [hash] [回退到指定hash号的版本]
    • git reflog 指针理解:ref(索引) log(日志) 查看指针指向的日志
12. 回到旧版本(回到旧版本的同时不删除新的版本)
  • 命令
    • git log
    • git checkout [hash] – [filename]/. [将某个/所有文件回退到hash值指定的版本并保留中间错误的版本]
13. 分支
a. 建立切换删除分支（在不同的版本之间切换时要确保原分支 working tree clean）
  • 命令
    • git branch /[name] [显示/创建分支]
    • git checkout [branch name] [切换分支]
    • git checkout -b [branch name] [创建并切换分支]
    • git branch [name] -d [删除分支][删除分支前需要merge]
    • git branch [name] -D [强制删除分支]
      tips:误删master分支时
git branch master [hash]
重新创建master分支，得到的结果和原来的一致
      b. 理解分支
  • 分支概述
    • 什么是分支：重新分配一块存储空间，把原来的分支中的内容完整的拷贝一份，包括hash值、log日志等
    • 分支的作用是什么：在重新创建的分支中可以进行实验性质的操作而不影响原有分支，如添加一个新的功能，测试成功后再合并到原有分支中
c. 合并分支
  • 命令
    • git merge [branch name] [在主分支中合并其他的分支，使得主分支拥有其他分支的功能]
d. 解决merge时的冲突问题
  • 解决冲突
    • git merge [branch name]
    • git status [查看冲突原因]
    • git merge –abort [忽略合并]
    • 手动选择正确的内容
    • git commit
