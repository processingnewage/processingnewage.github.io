类和对象

面向对象的三大特性：封装、继承、多态

1. 封装

  1. 意义：
    • 将属性和行为作为一个整体，表现生活中的事物
    • 将属性和行为加以权限控制
      • public : 公共权限: 类 内外均可访问
      • protected : 保护权限: 类中可访问, 类外不可访问, 可继承
      • private : 私有权限: 类中可访问, 类外不可访问, 不可继承
  2. struct 和 class 的区别 : 默认的访问权限不同
  3. 属性和方法称为成员, 分别叫做 成员属性/变量 和 成员函数/方法
  4. note: 一般将成员属性设为私有, 可以自己控制读写权限, 对于写权限, 我们可以检测数据的有效性, 同时在公有函数中添加set和get方法

2. 构造函数和析构函数

  1. C++中的面向对象来源于生活, 每个对象也都会有初始设置 以及 对象销毁前的清理数据的设置
  2. 一个对象或者变量没有初始状态, 对其使用后果是未知; 同样的使用完一个对象或变量, 没有及时清理, 也会造成一定的安全问题
  3. c++利用了构造函数和析构函数解决上述问题, 这两个函数将会被编译器自动调用, 完成对象初始化和清理工作
  4. 对象的初始化和清理工作是编译器强制要我们做的事情，因此如果我们不提供构造和析构，编译器会提供, 编译器提供的构造函数和析构函数是空实现
    • 构造函数: 主要作用在于创建对象时为对象的成员属性赋值，==构造函数解决了结构体需要初始化的难点==, 构造函数由编译器自动调用，无须手动调用
    • 析构函数: 主要作用在于对象销毁前系统自动调用，执行一些清理工作
  5. 构造函数和析构函数的注意事项:
    • 构造函数: 有参可重载, 程序在调用对象时候会自动调用构造, 无须手动调用,而且只会调用一次
    • 析构函数: 无参不重载, 程序在对象销毁前会自动调用析构, 无须手动调用, 而且只会调用一次
  6. ==函数重载== : 函数名相同, 可提高程序的复用性
    • 条件: 同一个作用域下, 名称相同, 但函数 参数类型不同 或者 个数不同 或者 顺序不同, 注意: 函数的返回值不可以作为函数重载的条件 [调用方式相同]
    • 注意事项:
      • 引用作为重载条件: int &a 和 const int &a 是不一样的
      • 函数重载碰到函数默认参数: 传参时有歧义
  7. 构造函数的==分类== : 有参构造 和 无参构造 ; 普通构造 和 拷贝构造
  8. 拷贝函数的==调用== : 括号法, 显示法, 隐式转换法
  9. 拷贝构造函数的==调用时机== :
    • 使用一个创建完毕的对象来初始化一个新对象
    • 值传递的方式给函数参数传值
    • 以值的方式返回局部对象
  10. 构造函数==调用规则== :
    默认情况下，c++编译器至少给一个类添加3个函数
    • 1．默认构造函数(无参，函数体为空)
    • 2．默认析构函数(无参，函数体为空)
    • 3．默认拷贝构造函数，对属性进行值拷贝
      构造函数调用规则如下：
    • 如果用户定义有参构造函数，c++不在提供默认无参构造，但是会提供默认拷贝构造
    • 如果用户定义拷贝构造函数，c++不会再提供其他构造函数
  11. 深浅拷贝 :
    • 浅拷贝：简单的赋值拷贝操作
    • 深拷贝：在堆区重新申请空间，进行拷贝操作
    • 在成员属性中涉及到在堆区申请内存时, 使用拷贝构造创建的对象和原对象中指向的是同一块内存,在析构时会出现重复释放的问题
    • 解决方案: 改写拷贝构造函数利用深拷贝在堆区重新申请不一样的内存
  12. 初始化列表 : Person(int a, int b, int c) :m_A(a), m_B(b), m_C(c) {}
  13. 类对象作为类成员 :
    • 构造的顺序是 ：先调用对象成员的构造，再调用本类构造
    • 析构顺序与构造相反
  14. 静态成员
    静态成员就是在成员变量和成员函数前加上关键字static，称为静态成员
    静态成员分为：
    • 静态成员变量
      • 所有对象共享同一份数据
      • 在编译阶段分配内存
      • 类内声明，类外初始化
    • 静态成员函数
      • 所有对象共享同一个函数
      • 静态成员函数只能访问静态成员变量: 所有的对象共享一个函数,无法区分不同的对象之间的区别,所以只能访问静态成员变量

3. C++对象模型和this指针

  1. 成员函数和成员变量 分开存储, 只有非静态成员变量才属于类的对象上, 非静态 成员函数和 静态成员 都不属于对象的栈上
  2. this 指针 : 每一个非静态成员函数只会诞生一份函数实例，也就是说多个同类型的对象会共用一块代码
  3. c++通过提供特殊的对象指针，this指针，解决上述问题, this指针指向被调用的成员函数所属的对象
  4. this指针是隐含在每一个非静态成员函数内的一种指针, this指针不需要定义, 直接使用即可
  5. this指针的用途：
    • 当形参和成员变量同名时，可用this指针来区分
    • 在类的非静态成员函数中返回对象本身，可使用return *this
    • this指针实际上是一个指针常量，指向是不变的【Person *const this】
  6. 空指针访问成员函数
  7. const修饰成员函数
    • 常函数： 常函数修饰的实际上是this 指针,也就是this指针指向的内容是不变的，所以不可以修改成员属性，【const Person *const this - 所指向的内容是不变的】，成员属性声明时加关键字mutable后，在常函数中依然可以修改
    • 常对象：声明对象前加const称该对象为常对象，常对象只能调用常函数
  8. 友元函数
    • 在程序里, 有些私有属性也想让类外特殊的一些函数或者类进行访问, 就需要用到友元的技术
    • 友元的目的就是让一个函数或者类访问另一个类中私有成员
    • 友元的关键字为 friend
    • 友元的三种实现
      • 全局函数做友元
      • 类做友元
      • 成员函数做友元

4. 运算符重载

  1. 运算符重载概念：对已有的运算符重新进行定义，赋予其另一种功能，以适应不同的数据类型，主要针对自定义的数据类型
  2. 对于内置的数据类型，编译器知道怎么使用运算符，运算符和内置数据类型是绑定的，运算符重载旨在扩展运算符的用途
  3. 常量引用：const引用的目的是，禁止通过修改引用值来改变被引用的对象
    1）const int & e 相当于 const int * const e
    2）普通引用 相当于 int *const e
    3）当使用常量（字面量）对const引用进行初始化时，C++编译器会为常量值
    分配空间，并将引用名作为这段空间的别名（编译器的优化）
    4）使用字面量对const引用初始化后，将生成一个只读变量
  4. 加法运算符的重载：成员函数 和 全局函数
  5. 左移运算符的重载：只能利用全局函数，成员函数输出是cout/cin在右边
  6. 输入输出流 也是 链式编程
  7. 递增运算符重载：注意区分前置++ 和 后置++
  8. 赋值运算符重载：成员属性有在堆区申请内存时，涉及到深浅拷贝的问题，深拷贝解决问题：在赋值时重新在堆区申请内存
  9. 关系运算符重载：判断对象是否相等
  10. 调用运算符重载：（）【仿函数】

5. 继承

  1. 继承的优点：减少重复代码，子类或派生类 父类或基类
  2. 继承方式：public/protected/private，父类中私有成员属性不可继承，继承方式决定的是公共属性和保护属性在子类中的权限
  3. 父类中所有非静态的成员属性都会被子类继承下去，父类中私有的成员属性被编译器隐藏，访问不到但是会被继承下去，也就是说在求sizeof()子类大小时会包括从父类中继承过来的私有成员属性
  4. 父类的static变量和函数在派生类中依然可用，但是受访问性控制（比如，父类的private域中的就不可访问），而且对static变量来说，派生类和父类中的static变量是共用空间的，这点在利用static变量进行引用计数的时候要特别注意，static函数没有“虚函数”一说，因为static函数实际上是“加上了访问控制的全局函数”
  5. 继承中的构造与析构：父类的构造函数–>子类的构造函数–>子类的析构函数–>父类的析构函数
  6. 子类与父类中同名成员的处理：子类中直接访问，父类中加作用域访问；静态成员的访问和非静态成员的访问一致，增加了一种通过类名访问的模式
  7. 多继承
  8. 菱形继承
    菱形继承概念： 两个派生类继承同一个基类，又有某个类同时继承着两个派生类这种继承被称为菱形继承，或者钻石继承
    通过作用域可以访问父类的属性，但不能确定自己的属性
    虚继承：virtual，孙子类使用两个儿子类之中的属性，最终得到的值以后赋值的为准，儿子类中的属性虚继承父类，得到vbptr指针，指向后赋值的属性值
    class Animal {};
    class Sheep : virtual public Animal {};
    class Tuo : virtual public Animal {};
    class SheepTuo : public Sheep, public Tuo {};

6. 多态

  1. 多态：相同的东西可以有多种形态
  2. 静态多态：函数重载 和 运算符重载；对函数名和运算符名的复用
  3. 动态多态：派生类和虚函数实现runtime多态
  4. 静态·多态 和 动态多态的区别：
    • 静态多态实现编译时确定函数地址—早绑定
    • 动态多态实现运行时确定函数地址—晚绑定
  5. 函数重写：函数所有的特征全部相同
  6. 动态多态的实现：父类中的函数加上virtual关键字，变成虚函数，在编译时不能确定函数地址，在子类中重写该函数，使用父类引用或指针指向子类对象，子类就会调用子类中重写的函数而不会调用父类中的同名函数
