一、引言

​ 单向链表（又名单链表、线性链表）是链表的一种，其特点是链表的链接方向是单向的，对链表的访问要通过从头部开始，依序往下读取。

二、单链表

  • 一个单向链表的节点被分成两个部分。第一个部分保存或者显示关于节点的信息，第二个部分存储下一个节点的地址。单向链表只可向一个方向遍历。

  • 单链表（带头节点）逻辑结构示意图如下，当下一个节点为空的时候，该链表就结束了

​ 注意：是逻辑结构，前面说过，在内存中节点不是一个接一个的。

三、单链表的应用

考虑这样一个场景：使用带head头的单向链表实现水浒英雄排行榜管理；完成对英雄人物的增删改查操作

// 英雄人物节点定义
class HeroNode {
  int no;
  String name;
  String nickName;
  HeroNode next;	// 指向下一个节点，为空则代表该链表结束。其他的字段则是上面所说的数据了。

  • 增加新节点
    • 第一种方法：在添加英雄时，直接添加到链表的尾部
    • 第二种方法：在添加英雄时，根据排名将英雄插入到指定位置，如果有这个排名，则添加失败，并给出提示
  • 修改：根据no属性查找匹配节点，修改节点数据
  • 删除：根据no属性查找匹配节点，删除节点
  • 查找：遍历单链表
  • 打印：遍历输出

class SingleLinkedList {
    // 头节点，不保存任何数据，只是用来作为一个起始点
    private HeroNode head = new HeroNode(0, "", "");

    // 添加节点：找到当前的最后节点，在后面添加新的节点
    public void add(HeroNode node) {
        // 要遍历到 next 为 null 的时候，才能进行添加
        HeroNode temp = head;
        while (true) {
            // 找到链表的最后,就退出循环
            if (temp.next == null) {
                break;
            }
            temp = temp.next;
        }
        temp.next = node;
    }
 
    // 打印链表中数据
    public void list() {
        if (head.next == null) {
            System.out.println("链表为空");
            return;
        }
        HeroNode temp = head.next;
        while (true) {
            // 如果是链表的最后
            if (temp == null) {
                break;
            }
            System.out.println(temp);     // 输出结果为重写的toString方法的格式
            temp = temp.next;
        }
    }
}

  • 有序实现增加新的节点

添加思路如下：

  1. 首先找到新添加节点的位置，通过辅助变量temp，然后循环遍历查找
  2. 新的节点.next = temp.next（建立新节点和后面的节点之间的联系）
  3. temp.next = 新的节点（建立新节点与前面的节点的联系）

简单一点就是：通过遍历找到要插入的位置，然后改变next的指向

代码实现，在原有的基础上新增了一个添加方法，如下

// 添加节点：排序插入
   public void addByOrder(HeroNode node) {
       // 由于 head 变量不能动，动了就无法从头遍历了，使用辅助变量来完成我们的添加
       HeroNode temp = head;
       boolean exist = false;  // 添加的节点是否已经在链表中存在
       while (true) {
           if (temp.next == null) {
               // 如果是链表尾，则跳出循环
               break;
           }
           // 如果当前节点的next编号，大于目标节点编号，则找到；应该将目标节点添加到temp 与next 之间
           if (temp.next.no > node.no) {
               break;
           }
           // 如果他们相等，则表示链表中已经存在目标节点了
           if (temp.next.no == node.no) {
               exist = true;
               break;
           }
           // 没找到，则后移temp ，继续寻找
           temp = temp.next;
       }
       if (exist) {
           System.out.printf("准备插入的英雄编号 %d 已经存在，不能加入 \n", node.no);
           return;
       }
       // 把节点插入到temp和temp.next之间：先建立后面节点的连接，再断开原连接（不然后面的节点会找不到）
       node.next = temp.next;
       temp.next = node;
   }

  • 修改节点信息

代码实现，在原来的基础上新增一个修改/更新方法，如下

// 修改节点：根据属性找到匹配的节点，进行修改
   public void update(HeroNode newNode) {
       // 先判断链表是否为空
       if (head.next == null) {
           System.out.println("链表为空");
           return;
       }
       HeroNode temp = head.next;
       boolean exist = false;  // 是否找到要修改的节点
       // 借助于标志变量，在循环中可只判断遍历结果，在循环结束后进行相关的操作
       while (true) {
           // 如果是链表尾部
           if (temp == null) {
               break;
           }
           // 如果已找到
           if (temp.no == newNode.no) {
               exist = true;
               break;
           }
           temp = temp.next;
       }
       // 如果已找到，则修改信息
       if (exist) {
           temp.name = newNode.name;
           temp.nickName = newNode.nickName;
       } else {
           System.out.printf("未找到编号为 %d 的英雄", newNode.no);
       }
   }

  • 删除节点

如上图所示，思路如下：

  1. 先找到需要删除的这个节点的前一个节点temp（为什么要找到前一个？因为是单向链表，找目标节点就无法完成删除了）
  2. temp.next = temp.next.next
  3. 被删除的节点，如果没有其他的引用的话，会被垃圾回收机制回收

代码实现，在原来的基础上新增一个删除方法，如下

// 删除节点：根据属性找到匹配的节点，删除相应节点
   public void delete(int no) {
       if (head.next == null) {
           System.out.println("链表为空");
           return;
       }
       HeroNode temp = head;
       boolean exist = false;  // 是否找到要删除的节点
       while (true) {
           if (temp.next == null) {
               break;
           }
           if (temp.next.no == no) {
               exist = true;
               break;
           }
           temp = temp.next;
       }
       if (!exist) {
           System.out.printf("未找到匹配的编号 %d \n", no);
           return;
       }
       // 删除操作
       temp.next = temp.next.next;
   }

  • 求单链表中有效节点的个数（不包括头节点）

思路：直接循环统计就行

// 统计有效节点的个数：不统计头节点的个数
public int getLength(HeroNode head) {
    int count = 0;
    // 判断链表是否为空，为空返回0值
    if(head.next == null) {
        return 0;
    }
    // 临时变量，头节点不好改变
    HeroNode temp = head;
    while(temp.next != null) {
        count++;
        temp = temp.next;
    }
    return count;
}

  • 查找单链表中倒数第k个节点

思路分析：

​ 1. 获得该链表节点的个数size

​ 2. 从第一个节点循环开始，到size - index结束

// 查找倒数第k个节点
public HeroNode findLastIndexNode(int index) {
    int size = getLength(head);
    if(size == 0) {
        return null;    // 此时为空
    }
    
    // 校验index
    if(index <= 0 || index >= size) {
        return null;
    }

    // 查找倒数第k个节点
    HeroNode temp = head;
    for(int i = 0; i <= size - index; i++) {
        temp = temp.next;
    }
    return temp;
}

  • 单链表的反转

思路分析：

  1. 定义一个新的reverseHead节点
  2. 从原链表中依次取出节点，并始终添加到reverseHead的第一个节点
  3. 将原head节点的next指向reverseHead.next

如下图所示：

// 单链表反转：逆序单链表
public void reverseList() {
    // 如果当前节点为空或者只有一个节点，无需反转，直接返回(边界条件先写完)
    if(head.next == null ||head.next.next == null) {
        return;
    }

    // 定义辅助指针，帮助遍历原链表
    HeroNode cur = head.next;   // 定义指针指向当前节点
    // 在将cur节点移动到reverse链表后，后面的节点会丢失
    HeroNode next = null;       // 定义指针指向当前节点（cur）的下一个节点
    HeroNode reverseHead = new HeroNode(0, "", ""); // 定义新的链表头指针

    // 遍历原来的列表，每遍历一个节点，就将其取出，并放到新的链表的最前面
    while (cur != null) {
        next = cur.next;    // 暂时保存当前节点的下一个节点，后面需要使用
        cur.next = reverseHead.next;    // 将cur的下一个节点指向新的链表的最前端
        reverseHead.next = cur;     // 将新的头节点连接上后续节点
        // 第一个节点next指向null
        // 第二个节点next指向原节点1
        // 后续节点next指向前一个被放置的节点，能够实现节点逆序
        // 上述两步分别建立两边的连接，相当于是一个插入的操作
        cur = next; // cur后移
    }
    // 最后一步将head.next指向reverseHead.next, 实现单链表的反转
    head.next = reverseHead.next;   // 理解赋值操作的本质，每一个next都是指向一个确切的节点
}

  • 从尾到头打印单链表（方式1：反向遍历；方式2：栈）

思路分析：

  1. 反向遍历 ：使用前面翻转操作后，再打印；有一个问题：会破坏原链表的结构
  2. 栈：利用栈先进后出的特点

如上图所示：

  1. 入栈操作：数据往栈底压入
  2. 出栈操作：数据从栈顶弹出

将原链表遍历，依次将每个节点压入栈中，然后遍历栈打印即可

// 从尾到头打印单链表：利用栈先进后出的特点
public void reversePrint() {
    // 先判断链表是否为空或只有一个节点：直接打印
    if(head.next == null || head.next.next == null) {
        return;
    }

    Stack<HeroNode> stack = new Stack<>();
    HeroNode cur = head.next;
    // 遍历链表，获取每一个节点，入栈
    while(cur != null) {
        stack.push(cur);
        cur = cur.next;
    }
    // 打印栈
    while(!stack.isEmpty()) {
        System.out.println(stack.pop());
    }
}

  • 合并两个有序的单链表，合并之后的链表任然有序

要求：合并两个有序的单链表，合并之后的链表依然有序

迭代思路：分别比较两个链表的节点，按序添加到一个新的链表

// 合并两个有序的单链表
public static HeroNode mergeTwoLists(HeroNode l1, HeroNode l2) {
    HeroNode prehead = new HeroNode(0, "", "");
    HeroNode prev = prehead;
    while (l1 != null && l2 != null) {
        if (l1.no <= l2.no) {
            prev.next = l1;
            l1 = l1.next;
        } else {
            prev.next = l2;
            l2 = l2.next;
        }
        prev = prev.next;
    }

    // 合并后 l1 和 l2 最多只有一个还未被合并完，我们直接将链表末尾指向未合并完的链表即可
    prev.next = l1 == null ? l2 : l1;
    return prehead;
}

五、双向链表

1. 单向链表的缺点

从前面的练习题发现单向链表的以下问题：

  • 查找方向：只能是单向
  • 不能自我删除：需要靠辅助节点，要找到删除节点的上一个节点和删除节点，才能完成删除

而以上问题，双向链表：

  • 可以双向查找
  • 可以自我删除

2. 双向链表的概念

​ 双向链表，又称为双链表，是链表的一种，它的每个数据结点中都有两个指针，分别指向直接后继和直接前驱。所以，从双向链表中的任意一个结点开始，都可以很方便地访问它的前驱结点和后继结点。一般我们都构造双向循环链表。

表头为空，表头的后继节点为”节点10”(数据为10的节点)；”节点10”的后继节点是”节点20”(数据为10的节点)，”节点20”的前继节点是”节点10”；”节点20”的后继节点是”节点30”，”节点30”的前继节点是”节点20”；…；末尾节点的后继节点是表头。

双向链表删除节点

删除”节点30”
删除之前：”节点20”的后继节点为”节点30”，”节点30” 的前继节点为”节点20”。”节点30”的后继节点为”节点40”，”节点40” 的前继节点为”节点30”。
删除之后：”节点20”的后继节点为”节点40”，”节点40” 的前继节点为”节点20”。

双向链表添加节点

在”节点10”与”节点20”之间添加”节点15”
添加之前：”节点10”的后继节点为”节点20”，”节点20” 的前继节点为”节点10”。
添加之后：”节点10”的后继节点为”节点15”，”节点15” 的前继节点为”节点10”。”节点15”的后继节点为”节点20”，”节点20” 的前继节点为”节点15”。

3. 双向链表分析

双向链表的结构如上图所示，每个节点都有pre和next变量，所以它可以往前查找或则往后查找。

那么下面先分析下双向链表的：遍历、添加、删除操作思路，之后再去实现：

  • 遍历：和单向链表类似，只是可以双向遍历了

  • 修改：和单向链表一样的方式

  • 添加：默认添加到双向链表的最后一个节点（建立双向连接）

    1. temp.next = newNode
    2. newNode.pre = temp

  • 按序添加：按照节点排序添加：

    1. 从头遍历节点，
    2. 找到比目标大的节点：插入到该节点之前（升序）（建立双向连接）
    3. 如果已经存在相同编号的节点，则提示不允许添加

  • 删除

    如上图所示，双向链表可以自我删除：

    1. 遍历找到要删除的那个节点temp
    2. temp.next.pre = temp.pre
    3. temp.pre.next = temp.next

代码实现如下：

package linkedlist;

// 场景：使用双链表实现水浒英雄排行榜管理，实现增删改查
public class DoubleLinkedListTest {
    public static void main(String[] args) {
        // 实现节点的增删改查
        DoubleHeroNode hero1 = new DoubleHeroNode(1, "宋江", "及时雨");
        DoubleHeroNode hero4 = new DoubleHeroNode(4, "卢俊义", "玉麒麟");
        DoubleHeroNode hero7 = new DoubleHeroNode(7, "吴用", "智多星");
        DoubleHeroNode hero9 = new DoubleHeroNode(9, "林冲", "豹子头");

        DoubleLinkedList DoubleLinkedList = new DoubleLinkedList();
        DoubleLinkedList.add(hero1);
        DoubleLinkedList.add(hero4);
        DoubleLinkedList.add(hero7);
        DoubleLinkedList.add(hero9);

        DoubleLinkedList.print();
    }
}

class DoubleLinkedList {
    // 头节点，不保存任何数据，只是用来作为一个起始点
    public DoubleHeroNode head = new DoubleHeroNode(0, "", "");

    // 将节点添加到链表的尾部
    public void add(DoubleHeroNode node) {
        DoubleHeroNode temp = head;

        // 找到链表的末尾
        while (temp.next != null) {
            temp = temp.next;
        }
        // 将新节点添加到末尾的节点上：建立双向连接
        temp.next = node;
        node.pre = temp;
    }

    // 按照序号添加节点：和单链表实现是一样的
    public void addByOrder(DoubleHeroNode node) {
        // 先判断边界条件
        if(head.next == null) {
            return;
        }

        // 查找添加的节点的位置
        DoubleHeroNode temp = head.next;
        boolean isFound = false;      // 标识要查找的节点是否存在链表中
        while (temp != null) {
            // 编号相等的情况
            if(temp.no == node.no) {
                isFound = true;
                break;
            }
            // 找到了节点位置
            if(temp.no > node.no) { // 此时新插入的节点应该位于temp之前
                break;
            }
            temp = temp.next;
        }
        if(isFound) {
            System.out.println("该节点已存在...");
            return;
        }

        // 插入新节点：建立双向连接
        node.next = temp;
        node.pre = temp.pre;
        temp.pre.next = node;
        temp.pre = node;

    }

    // 更新节点：以no匹配节点，与单向链表的处理逻辑一致
    public void update(DoubleHeroNode node) {
        // 边界条件的分析：判断链表是否为空
        if(head.next == null) {
            System.out.println("链表为空");
            return;
        }

        DoubleHeroNode temp = head.next;
        boolean isExist = false;        // 标识在循环中是否找到该节点
        while (temp != null) {
            if(temp.no == node.no) {
                isExist = true;
                break;
            }
            temp = temp.next;
        }
        
        // 如果已找到，则修改信息
        if(isExist) {   // 实际上temp就是匹配得到的节点
            temp.name = node.name;
            temp.nickName = node.nickName;
        } else {
            System.out.println("未找到节点...");
        }
    }

    // 删除节点：根据no匹配，删除对应节点
    // 双向链表可以自我删除：直接找到该节点，然后自我删除
    public void delete(DoubleHeroNode node) {
        // 判断链表是否为空
        if(head.next == null) {
            System.out.println("该链表为空...");
            return;
        }

        DoubleHeroNode temp = head.next;
        boolean isExist = false;        // 标识查找结点是否存在
        while (temp != null) {
            if(temp.no == node.no) {
                isExist = true;
                break;
            }
            temp = temp.next;
        }

        // 如果节点不存在
        if(!isExist) {
            System.out.println("该节点不存在...");
            return;
        }

        // 节点存在，完成自我删除：temp节点就是匹配得到的节点
        // 这里有一个边界问题，当temp为末尾节点时，next为空，不需要处理
        if(temp.next != null) {
            temp.next.pre = temp.pre;
        }
        temp.pre.next = temp.next;      // 可以理解为节点之间的对应
    }

    // 打印链表：遍历链表然后打印输出
    public void print() {
        // 判断是否为空链表：边界值条件的分析
        if(head.next == null) {
            return;
        }

        DoubleHeroNode temp = head.next;
        while(temp != null) {
            System.out.println(temp);
            temp = temp.next;
        }
    }
}

// 节点定义
class DoubleHeroNode {
    public int no;
    public String name;
    public String nickName;
    public DoubleHeroNode next;   // 默认初始化为null
    public DoubleHeroNode pre;   // 默认初始化为null

    // 构造器
    public DoubleHeroNode(int no, String name, String nickName) {
        this.no = no;
        this.name = name;
        this.nickName = nickName;
    }

    @Override
    public String toString() {
        return "HeroNode{" +
                "no=" + no +
                ", name='" + name + '\'' +
                ", nickName='" + nickName + '\'' +
                '}';
    }
}

六、环形链表(循环链表)

1. 单向环形链表

应用场景-约瑟夫问题

约瑟夫(Josephu)问题，也就是丢手帕问题，他的规则如下

  • 有编号为1~n的n个人围坐在一起
  • 约定编号为K(1<=k<=n)的人从1开始报数
  • 数到m的那个人出列，它的下一位又从1开始报数

循环以上过程，直到所有人都出列，并列出出列人的编号。

该问题其实可以使用单循环链表(单向环形链表)来解决，思路如下：

  1. 先构成一个有n个节点的单循环链表
  2. 然后由k节点起从1开始计数
  3. 计数到m时，对应节点从链表中删除，然后从下一个节点又从1开始计数

循环以上过程，直到最后一个节点从链表中删除，算法结束

单向环形链表介绍

单向环形链表的的逻辑结构如下，所有的节点之间形成了一个环状。

约瑟夫问题示意图

需求如下：

1. `n=5`：有 5 个人
2. `k=1`：从第一个人开始数
3. `m=2`：数两次

没有动图，那么使用下面的描述来讲解：

  1. 第一轮：2出队列，1.next=3

    还剩下：1、3、4、5

  2. 第二轮：4出队列，3.next=5；（从3开始报数，第2个的出队列，也就是4）

    还剩下：1、3、5

  3. 第三轮：1出队列，5.next=3

    还剩下：3、5

  4. 第四轮：5出队列，3.next=3

    还剩下：3，自己的next就是自己

  5. 第五轮：3出队列，队列中无元素，结束

那么最终的出队列顺序就是：2、4、1、5、3

约瑟夫问题可以使用数组来解决，这里使用单向环形链表，比较好理解

// 单向环形链表实现：构建循环链表
class CircleSingleLinkedList {
    public Boy first; // 默认为null的第一个节点(在不带头节点的链表中代替头节点的作用)

    // 构建循环链表：传入构建的链表的长度(在构建过程中运用多个指针)
    public void add(int nums) {
        if (nums < 1) {
            System.out.println("至少添加一个节点...");
            return;
        }

        Boy temp = null; // 可以移动的辅助变量
        for (int i = 1; i <= nums; i++) {
            Boy boy = new Boy(i); // 创建新的节点
            // 初始化first节点
            if (first == null) {
                first = boy; // first指向第一个节点并在以后的过程中保持不动
                temp = first; // 保存当前节点
                boy.next = boy; // 构成环状
                continue; // 中断此次循环继续执行下一次循环
            }

            temp.next = boy;
            boy.next = first;
            temp = boy;
        }
    }

    // 打印输出循环链表
    public void print() {
        // 判断链表长度
        if (first == null) {
            System.out.println("该链表为空...");
            return;
        }

        // 当链表不为空时
        Boy temp = first; // first节点不移动，只能定义辅助节点代替
        while (temp.next != first) {
            System.out.println(temp);
            temp = temp.next;
        }
        System.out.println(temp);
    }

    /**
     * 游戏开始，计算出圈顺序
     *
     * @param startNo  从第几个小孩开始数
     * @param countNum 数几下
     * @param nums     参与该游戏的小孩有多少个
     */
    public void countBoy(int startNo, int countNum, int nums) {
        // 进行数据校验
        if (first == null || // 环形链表没有构建
                countNum < 1 || // 每次至少数一下
                startNo > nums // 开始小孩不能大于参加游戏的人数
        ) {
            System.out.println("参数有误，请重新输入：");
        }

        // 1. 初始化辅助变量的位置：在first的后面一个位置（first的位置不需要固定）
        Boy temp = first;
        while (temp.next != first) {
            temp = temp.next;
        }

        // 2. 定位到开始数字的位置（将first定位到开始节点，temp为first前一个节点）
        for (int i = 0; i < startNo - 1; i++) { // 移动次数为startNo - 1
            temp = temp.next;
            first = first.next;
        }

        // 为了测试方便，这里添加一个日志输出
        System.out.printf("定位到位置： %d \n", startNo);
        print();

        // 3. 开始报数和出圈
        while (true) {
            // 当队列中只剩下一个人的时候，跳出循环，因为最后一个必然是他自己出队列
            // 其实不用这段代码也可以，只是first和temp指针会在最后一个节点上循环跳转
            if (temp == first) {
                break;
            }
            // 报数：每次报数 m-1
            for (int i = 0; i < countNum - 1; i++) {
                // 因为temp永远在first后面，只要在first移动时，指向first原来所在的位置即可
                temp = first;
                first = first.next;
            }
            // 出圈
            System.out.printf("出圈小孩编号 %d \n", first.id);
            first = first.next; // first重置为下一次报数的小孩节点上
            temp.next = first; // temp重置为下一次报数的小孩节点的前一个节点上
        }
        System.out.printf("最后剩下小孩编号 %d \n", first.id);
    }
}

// 节点实现
class Boy {
    int id;
    Boy next; // 指针域，指向下一个节点

    // 构造器
    public Boy(int id) {
        this.id = id;
    }

    // 重写toString()
    @Override
    public String toString() {
        return String.valueOf(id);
    }
}
