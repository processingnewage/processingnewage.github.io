编译过程：

  • 预处理（preprocessing）
  • 编译（compile）
  • 链接（link）
  • 执行（execute）

头文件中包含函数的声明部分，而具体的实现在其他的cpp源文件中；

因此预处理部分包含头文件相当于包含函数声明；

计算机基础架构（冯诺伊曼架构）

  • 一种将程序指令存储器和数据存储器合并在一起的存储器结构，因此程序指令和数据的宽度相同
  • 计算机的三个基本原则：
    • 二进制逻辑
    • 程序存储执行
    • 计算机由五个部分组成

  • 冯诺依曼架构的特点：

    • 单处理机结构，机器以运算器为核心
    • 采用程序存储思想
    • 指令和数据一样可以参与运算
    • 数据以二进制形式表示
    • 软件和硬件完全分离
    • 指令由操作码和操作数组成
    • 指令顺序执行
    • 哈佛架构

  • 编译过程：

    • 预处理（Preprocessing）：g++ -E file.cpp -o file.i
    • 编译（Compilation）：g++ -S file.i -o file.s
    • 汇编（Assembly）：g++ -c file.s -o file.o
    • 链接（Linking）：g++ file.o -o file(在Windows和Linux下生成可执行文件)

指针（指针是一个类型）

  • 指针类型的变量，其值是另一个变量的地址
  • 指针变量声明的一般形式为：type* pointerVarName
  • 指针的大小：
    • 指针类型的大小是固定的（无论该指针指向哪种数据类型），在32位系统中为4字节，在64位系统中为8字节；
    • 对指针类型的变量进行加减操作时，实际上是对该指针变量中保存的地址进行加减操作，所以这些加减操作体现在内存地址上，就与指针指向的具体数据类型有关了。例如在本例中，对指向char类型的指针s进行+1操作，则体现在内存地址上的结果就是地址+1，因为指针s指向的char类型在内存中占用1个字节；同理，对指向int类型的指针a进行+1操作，体现在内存地址上的结果就是地址+4，因为指针a指向的int类型在内存中占用4个字节（int类型在32和64位环境下内存都是占用4个字节，所以+4）；指针
    • 类型本身的大小，与对指针进行的加减操作无关，两者不要混淆。
  • 对指针类型、变量、值的辨析

内存模型

  • 堆：这里保存的数据只是为了临时存储一些值而创建的，而我们可能在程序运行过程中可能会回收此内存。因为我们在程序执行期间不需要很长时间，所以使用C中的new或malloc这类内存分配程序来为我们所需的特定数据类型提供新的空间，并且随着我们要求越来越多的动态数据空间而该区域不断扩大，并且在内存中逐渐增长到更高的地址。

  • 栈：当我们执行这些过程调用时,堆的基本特性是LIFO,存储着该程序“上下文”，它将从内存的高层地址开始，然后向另一个方向向下扩展。上下文其实就是程序中各个函数之间调用的先后顺序。

  • 其他

    • 代码段：程序的所有指令会存放在这个区域，这是已经编译后的机器码；
    • 字面量池：是程序初始化时的一些字符串字面量，在程序中用于显示文字
    • 全局数据段：程序初始化时的常量和全局/静态的变量，C/C++ 用global/static声明的变量都存放在这个区域，对所有函数公开可见。

#### **引用（reference）**

>C++强大之处在于可以直接指定变量的类型是引用、指针还是其他类型

* 引用是一个类型（指针和引用的类型标识所指向的变量/常量的类型，有利于类型安全）

* 来定义一个新的变量是一个已存在的变量的别名（alias）（同样的内存地址）

* 一旦把引用初始化为某个变量，就可以使用该引用名称来操作变量

* 注意区分指针和引用

* 引用的特点：

  * 引用的内存意义：指向同样的地址
  * 引用对传参具有重要的意义，不再需要拷贝值（值和地址值）
  * 不存在空引用，引用必须连接到一块合法的地址
  * 一旦被初始化为一个对象，就不能被指向另一个对象；指针可以在任何时候指向另一个对象
  * 引用必须在创建的时候被初始化；指针可以在任何时候被初始化

指针常量和常量指针

  • 指针常量即指针类型的常量：其值（所指向变量的地址值）不可改变；即为引用的概念

  • 常量指针即指向常量的指针：其指向一个常量，该常量的值不可改变，但指针的指向随时可以改变

    例子：

    ​ 在栈上有一个 const int data = 2;

    ​ 定义一个常量指针p = &data; p可随时改变指向，但data的值不可改变

    ​ 在栈上有一个 int value = 3;

    ​ 定义一个指针常量q = &value; q的值（即为value变量的地址值）不可改变，即q的指向不可改变，但value的值随时可以改变

  • 引用即对指针常量使用过程的封装：

    • 在定义时使用&代替了指针常量的定义过程

      const int* q = &data;

      int& q = data;

    • 在使用时代替了解引用的过程

      int value = &q; 解引用

      int value = q;

      ​ 局部变量的作用域问题：每个函数调用都有自己独立的栈帧（frame），这就是每个函数独立的作用域，其中的局部变量属于这个作用域之中，不同作用域中的局部变量的地址值是不同的（这也是为什么在不同的函数中可以命名相同的变量名的原因），但需要注意的是，不同函数作用域中的局部变量的内存分配仍然是在栈区（以6开头的地址）

      关于函数栈帧知识的补充：

      ​ 内存栈区 (stack) 由操作系统自动分配和释放，用于存放函数的形参值、局部变量的值、函数返回地址等数据，其操作方式与数据结构中的栈一致，都是后进先出的原则。在虚拟内存地址空间中，栈从高地址向低地址延伸。

      ​ 栈帧 (stack frame) 是编译器用来实现函数调用的一种数据结构，是内存栈区的基本单元。内存栈空间上保持了 N 个栈帧的实体。（栈帧的创建应该是在堆区）

      ​ 所有函数调用均发生在栈上，每个函数的每次调用，都有它自己独立的一个栈帧。寄存器 ebp 指向当前栈帧的底部 (高地址)，寄存器 esp 指向当前栈帧的顶部 (低地址)。
