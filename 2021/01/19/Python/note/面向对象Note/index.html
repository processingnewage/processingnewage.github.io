1. 面向对象的编程思想

  • 面向对象是一种编程思想，将相关的数据和动作聚合在一起，达到解耦合、提高程序结构的清晰性的目的，而不只是具体的语法格式。“整合” “抽象”（彩妆盒的道理，整合了工具和原料，拿到彩妆盒就可以化妆了）。

  • 列表、字典、文件等等也是对象，但使用起来并不方便，所以python提供了新的语法格式来定义类与对象。

  • 程序中使用的是一个个的实例对象，类即是归类的意思，是对相似的对象共同的部分的抽象，可与节约声明对象的存储空间。

  • 对象是“容器”：存放数据和功能

  • 类也是“容器”：存放同类对象共有的数据和功能

  • 所以：类 也是 对象

  • 类的命名使用驼峰体 （大驼峰）

    补充：**python的标识符命名规范**
* 简单地理解，标识符就是一个名字，就好像我们每个人都有属于自己的名字，它的主要作用就是作为变量、函数、类、模块以及其他对象的名称
* Python中标识符的命名不是随意的，而是要遵守一定的命令规则，比如说：
  * 标识符是由字符（A~Z 和 a~z）、下划线和数字组成，但第一个字符不能是数字
  * 标识符不能和 Python 中的保留字相同
  * Python中的标识符中，不能包含空格、@、% 以及 $ 等特殊字符
* 标识符中的字母是严格**区分大小写**的
* 以下划线开头的标识符有特殊含义
  + 以单下划线开头的标识符（如 _width）（受保护的），表示不能直接访问的类属性，其无法通过 from...import* 的方式导入
  + 以双下划线开头的标识符（如__add）表示类的私有成员
  + 以双下划线作为开头和结尾的标识符（如__init__（）），是专用标识符
  因此，除非特定场景需要，应避免使用以下划线开头的标识符
  
* 标识符的命名，除了要遵守以上这几条规则外，不同场景中的标识符，其名称也有一定的规范可循
  + **模块名**：应尽量短小，并且全部使用小写字母，可以使用下划线分割多个字母，例如 game_mian、game_register 等
  + **包**：应尽量短小，也全部使用小写字母，不推荐使用下划线，例如 com.mr、com.mr.book 等
  + **类**：应采用单词首字母大写的形式。例如，定义一个图书类，可以命名为 BookGiven
  + **模块内部的类名**：可以采用 "下划线+首字母大写" 的形式，如 _Book;
  + **函数名、类中的属性名和方法名**：应全部使用小写字母，多个单词之间可以用下划线分割
  + **常量**：命名应全部使用大写字母，单词之间可以用下划线分割
  + **变量**：小写字母加下划线的形式，例如 text_input、given_push等

  • 类体代码在定义阶段就运行，类是对象相似数据和功能的集合体，所以类体中最常见的是变量与函数的定义，但是类体中可以包含任意其他的代码。会产生类的namespace。（所以类的namespace是在定义阶段就产生的，即类在定义阶段即运行）

  • 调用类生成对象，类体在定义阶段执行过了，类的调用不是执行类体代码，与函数有本质的区别。使用类去创建对象是建立类与对象之间的关联。

  • 在创建对象后，对对象自身的数据和行为的定义可以使用字典或是属性访问的方式实现（属性访问的本质还是字典的操作）；但这两种方式实现起来较为复杂，在python中__init__()函数可用于接受参数的传递直接实现对对象独有属性的赋值，提升了编程的简便性。（init方法中赋值的属性属于实例对象所有）

  • __init__()方法在类的定义阶段不会运行，在类的调用阶段自动触发，传入对象名称作为初始化函数的第一个参数。

  • 调用类的过程称之为实例化：

    • 生成一个空对象
    • 自动调用初始化方法，传入空对象以及参数，为对象属性赋值
    • 返回初始化的完整对象

  • 总结__init__()方法：

    • 初始化方法内存放的是为对象初始化属性的功能，但是也可存放任意的其他代码，想要在类调用时立刻执行的代码都可以放到该方法内部

  • 属性查找：

    • 对象的名称空间里只存放着对象独有的属性，而对象们相似的属性是存放于类中的。对象在访问属性时，会优先从对象本身的dict中查找，未找到，则去类的dict中查找
      • 类中定义的变量是类的数据属性，是共享给所有对象用的，指向相同的内存地址。修改类的变量时相当于函数中的全局变量，所有的与该类相关的对象访问该变量的值均有变化；修改对象的变量时，类似于字典的操作，若修改的是与类中的同名变量，则新创建该变量，属于该对象的namespace（类似于函数中的局部变量和全局变量）
      • 类中定义的函数是类的函数属性，类可以使用，但必须遵循函数的参数规则，有几个参数需要传几个参数，但其实类中定义的函数主要是给对象使用的，而且是绑定给对象的，虽然所有对象指向的都是相同的功能，但是绑定到不同的对象就是不同的绑定方法，内存地址各不相同
      • 绑定到对象的方法特殊之处在于，绑定给谁就应该由谁来调用，谁来调用，就会将’谁’本身当做第一个参数自动传入（方法__init__()也是一样的道理）：self指针
      • 绑定方法：直接通过类调用函数不够简便，使用对象的方式调用类中定义的共同的函数属性，使用self指针来区分每个不同的对象，不同的对象之间使用的是相同的函数功能，但是是不同的绑定方法，指向的是同一段代码

2. 面向对象的三大特性

  • 封装：

    • 封装指的就是把数据与功能都整合到一起，听起来是不是很熟悉，没错，我们之前所说的”整合“二字其实就是封装的通俗说法。除此之外，针对封装到对象或者类中的属性，我们还可以严格控制对它们的访问，分两步实现：隐藏与开放接口

    • 隐藏：数据隐藏 和 函数隐藏

# 总结：
	1. python中使用__name的方式隐藏属性，但实际上只是做简单的变形操作，在类的字典中可以查看到：__name变形为_Foo_name，使用Foo._Foo__name仍然可以查看到该属性；所以python的隐藏不是真正的隐藏。
    2. 属性的隐藏对外不对内，__开头的属性在检查类内代码语法时统一发生变形，因为类在定义阶段即运行，所以类内代码实际运行时已经是变形后的语句了（__属性名__不隐藏，表示内置属性）
    3. 变形操作只在类内（类体）检查语法的时候发生一次，之后定义的__开头的属性都不会变形（类的设计者事先决定好哪些属性需要隐藏，在类外定义的符合隐藏格式的属性值并不会被隐藏）
    4. 为什么需要隐藏：
    	1. 隐藏数据属性：限制类外部对数据的直接操作，类内可以通过相应的接口操作数据，接口之上可以附加额外的逻辑对数据进行严格的控制（例如可以检查用户的输入合法性）
        2. 隐藏函数属性：隔离复杂度，有些函数并不需要提供给外部用户使用，隐藏在类内，由统一的接口调用有利于程序更加的清晰
    5. 总结隐藏属性与开放接口，本质就是为了明确地区分内外，类内部可以修改封装内的东西而不影响外部调用者的代码；而类外部只需拿到一个接口，只要接口名、参数不变，则无论设计者如何改变内部实现代码，使用者均无需改变代码。这就提供一个良好的合作基础，只要接口这个基础约定不变，则代码的修改不足为虑

  • 继承

    • 什么是继承：继承是一种新建子类的方式，新建的类称为子类或派生类，父类称为基类或超类，子类可以遗传父类的特性，实现代码的重用

      • 需要注意的是：python支持多继承，即新建的类可以继承一个或多个父类，多继承的优点和缺点：

      • 优点：子类可以同时遗传多个父类的属性，最大限度的重用代码

      • 缺点：

        • 违背人的思维习惯（子类同时属于多个父类）
        • 代码的可读性变差
        • 代码的扩展性变差，不建议使用多继承，可能引发菱形继承问题（不可避免地涉及到多继承时，应该使用Mixins机制）

    • 菱形继承：一个子类继承的多个父类最终汇聚到一个非object类

    • 继承原理：python到底是如何实现继承的呢？ 对于你定义的每一个类，Python都会计算出一个方法解析顺序(MRO)列表，该MRO列表就是一个简单的所有基类的线性顺序列表

      # 对于多继承的问题时，考虑较为复杂的MRO列表的情况，而对于大多数单继承的应用情况，只做简单的考虑，不需要去分析继承方法解析顺序的问题
    python会在MRO列表上从左到右开始查找基类，直到找到第一个匹配这个属性的类为止，而这个MRO列表的构造是通过一个C3线性化算法来实现的，我们不去深究这个算法的数学原理，它实际上就是合并所有父类的MRO列表并遵循如下三条准则:
	1. 子类会先于父类被检查
	2. 多个父类会根据它们在列表中的顺序被检查
	3. 如果对下一个类存在两个合法的选择，选择第一个父类  

# 总结：
	1. 由对象发起的属性查找，会从对象自身的属性里检索，没有则会按照对象的类.mro()规定的顺序依次找下去
	2. 由类发起的属性查找，会按照当前类.mro()规定的顺序依次找下去
    • 深度优先和广度优先
    # 总结：
	1. 多继承结构为非菱形结构，最后寻找object类
    2. 多继承关系为菱形结构，那么经典类与新式类会有不同MRO，分别对应属性的两种查找方式：深度优先和广度优先（区分：是否先访问最高层级的父类）
    • Python Mixins机制
    多继承的正确打开方式
	Mixins机制：在多继承背景下尽可能地提升程序的可读性，并且满足人类的思维习惯
    大多数编程语言只能实现单继承，满足父类与子类之间“is a”的思维习惯，准确的表达子类的类别；
    Mixins机制使用的是指定的编程规范，子类只属于一个确定的父类（确定其类别），其他的使用Mixins类实现某些功能，Mixins类应该即插即用，实现功能相对单一（只是一种编程规范，类似于常量的命名）

        # 传入当前类和self，调用super函数得到一个特殊的对象，该对象会参照发起属性查找的类的MRO，按照MRO的顺序寻找属性
        super().__init__(name, age, sex)

# 2. 自定义元类来控制类的生成
class Mymeta(type): # 只有继承了type类才能称之为一个元类，否则就是一个普通的自定义类
    # 需要自定义功能时重写方法，否则直接继承type元类中的call方法即可
	def __call__(self, *args, **kwargs):
		# 1. Mymeta.__call__()会先调用People的__new__()
		people_obj = self.__new__(self)
		print(people_obj.__dict__)	# 输出无属性
		# 2. Mymeta.__call__()然后调用People的__init__()
		self.__init__(people_obj, *args, **kwargs) # 初始化
		print(people_obj.__dict__) # 初始化后输出有属性，说明获取传值的过程是在init()方法中实现的
		# 3. Mymeta.__call__()最后返回一个初始化好的对象
		return people_obj

class Person(object, metaclass=Mymeta):	# 默认metaclass = type
	def __init__(self, name, age):
		self.__name = name
		self.age = age

	def say(self):
		print("{}: {}".format(self.__name, self.age))
    
obj = Person("egon", 11)
