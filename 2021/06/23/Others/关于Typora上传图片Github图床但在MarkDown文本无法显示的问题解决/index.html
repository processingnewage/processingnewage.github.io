问题描述

​ 使用 Typora写 MarkDown笔记，先用 picgo将图片上传到 Github仓库中，上传成功了，在 Github中也能看到图片在仓库里面。

网络正常，打开 Github是可以的，但是在 MarkDown中引用这个图片，却显示加载失败，该怎么解决 ？

问题解决

​ 你是不是到了 Github图片所在的地址然后直接拷贝了URL粘贴上去的？我一开始也是这样做的，后来发现一件事：还是在 Github图片所在的主页，右键点击图片——复制图片地址，你粘贴出来对比一下一开始你拷贝的URL发现后边多了个参数

比如原来我的是

https://github.com/StarkSchroedinger/StarkSchroedinger.github.io/blob/master/img/in-post/2020-09-25-Lie-Group/RotateVector.png

复制图片地址后的结果是

https://github.com/StarkSchroedinger/StarkSchroedinger.github.io/blob/master/img/in-post/2020-09-25-Lie-Group/RotateVector.png?raw=true

对比一下，就是后边那个“?raw=true”不一样，加上那个就可以了。

​ 也就是说不要直接拷贝URL，那个URL是图片在 Github上显示的地址。而在 MarkDown上或者其他地方，你其实是为了展示图片向 Github后台作出的一个请求，而”?raw=true”就是这个请求的一个参数，也就是说你不加这个参数请求的相当于是图片所在的 Github上的页面，而不是图片本身。
